**Git** git ======================================================= ## Workflow ![Git workflow](images/git.svg) ### HEAD ~~~ HEAD~n # n世代前の親 [HEAD~~] --- [HEAD~] --- [HEAD] [HEAD~2] HEAD^n # n番目の親 [HEAD^^] --- [HEAD^ ] -+- [HEAD] | [HEAD^2] -+ ~~~ ### make a remote repository ~~~ mkdir myproj.git cd myproj.git git init --bare ~~~ ### make a local repository - URL=https://path/myproj.git - URL=X:/path/myproj.git #### clone ~~~ git clone URL [work_dir] git clone -b branch_name URL # specify branch git clone --recurse-submodules URL # recursive ~~~ #### init ~~~ mkdir myproj cd myproj git init git remote add origin URL ~~~ ### add, commit, push ~~~ git add {files} # stage files git commit -m "message" # commit changes to the local repository git push origin master # push the "master" branch to the "origin" remote repository ~~~ ### discard changes ~~~ git checkout [--] {file} # discard chenges of a file git reset --hard # discard all changes ~~~ ### add / reset files ~~~ git add {files} # stage git reset {files} # unstage ~~~ ### commit / reset --soft ~~~ git commit -m "message" git reset --soft HEAD^ # reset the last commit ~~~ ### track / untrack files ~~~ git add {files} git rm --cached {file} # untrack file git rm --cached -r {dir} # untrack dir git ls-files # list tracked files git ls-files -o --directory # list untracked files ~~~ ### create / delete a branch ~~~ git branch {branch} # create a branch git branch -d {branch} # delete a branch ~~~ ### push / delete a branch to a remote repository ~~~ git push origin {branch} # push a branch to a remote repository git push origin :{branch} # delete a branch to a remote repository ~~~ ### create / delete a tag ~~~ git tag {tag} # create a tag git tag -d {tag} # delete a tag ~~~ ### push / delete a tag to a remote repository ~~~ git push origin {tag} # push a tag to a remote repository git push origin :{tag} # delete a tag to a remote repository ~~~ ### chmod (Windows) ~~~ git update-index --add --chmod=(+|-)x {file} # add|remove exec permission git ls-files -s # show filemode of tracked files git config -l | grep filemode # show filemode setting ~~~ ## config ~~~ git config --global user.name {name} git config --global user.email {email} git config --global core.autocrlf input git config -l # list config ~~~ ## information ~~~ git status git status -sb # show status in short format git log # show commit log git reflog # show reflog git ls-files -s # list tracked files with filemode git ls-files -o --directory # list untracked files git branch -a [-vv] # list local and remote branches [verbose] git remote -v # list remote repositories git rev-parse --show-cdup git rev-parse --abbrev-ref HEAD # branch name git rev-parse --short HEAD # commit hash git symbolic-ref --short HEAD # branch name (git > 1.8) git branch --show-current # branch name ~~~ ## remote repository ~~~ git remote add origin URL # connect oring to URL git remote set-url origin URL # set origin to URL git remote rename {src} {dst} # rename git remote rm origin # disconnect origin git remote prune origin # prune remote branches git remote show origin # show info of the remote repository git remote -v # list remote repositories ~~~ ## remote branch ~~~ git fetch git fetch --prune # prune remote branches git pull origin {branch} # fetch + merge git push origin {branch} git push origin :{branch} # remove the remote branch ~~~ ## add ~~~ git add {files} # ファイルをtraced fileにする git add . # カレントディレクトリのすべてのファイルをtraced fileにする git add -u # add updated files ~~~ ## reset ~~~ git reset [tree-ish] [--] {files} # unstage git reset --hard [commit] # reset [commit, add, work] git reset [--mixed] [commit] # reset [commit, add] git reset --soft [commit] # reset [commit] ~~~ ## commit ~~~ git commit git commit -a # 変更されたtraced fileをaddしてcommit git commit -m "message" # メッセージをつけてコミット git commit --amend --author="name" # 名前を変えてコミットしなおす git commit --amend -m "message" # コミットメッセージを変更 ~~~ ## commit message ~~~ 一行目は概要 二行目は空行 三行目以降に詳細 - 現在形で記述する ~~~ ## checkout co ~~~ git checkout {branch} # ブランチへ移動 git checkout -b {branch} # ブランチを作成して移動 git checkout -b {branch} {origin/branch} # リモートブランチをチェックアウト git checkout [--] {files} # ファイルをリポジトリから取り出す -- 以降はファイル名として扱う (ファイル名がブランチ名と被るのを避ける) git checkout --orphan {branch} # 空ブランチを作成 ~~~ ## branch ~~~ git branch {branch} # ブランチの作成 git branch -d {branch} # ブランチの削除 git branch -m {branch} # 現在のブランチ名の変更 git branch -a # ブランチの表示 ~~~ ## tag ~~~ git tag {name} # タグの作成 git tag -d {name} # タグの削除 git tag # タグ一覧 git show {name} # タグ情報 git push origin {name} # タグをリモートに反映 git push origin --tags # 全てのタグをリモートに反映 git push origin :{name} # リモートタグ削除 git ls-remote --tags # リモートタグ一覧 ~~~ ## merge ~~~ git merge {branch} # 指定したブランチを現在のブランチに統合する git merge {branch} --no-ff # always create a merge commit git merge --continue # after resolving conflicts git merge --abort # cancel git checkout --ours {file} git checkout --theirs {file} [master] --o--H \ [branch] o--B git merge B [master] --o--o---H \ / [branch] o--B ~~~ ## rebase ~~~ git rebase {branch} # 現在のブランチの派生元を指定したブランチに変更する git rebase -i {branch} # コミット編集 git rebase -i {commit} # コミット編集 git rebase --continue # after resolving conflicts git rebase --abort # cancel [master] --o--M \ [branch] o--H git rebase M [master] --o--M \ [branch] o--H ~~~ ### [example] rebase -i {commit} でコミットを１つにまとめる (squash) ~~~ [master] --C--M \ [branch] 1--2--H \ / 3-4 git rebase -i C pick 1 pick 2 pick 3 pick 4 # pick を squash に変更 pick 1 s 2 s 3 s 4 [master] --C--M \ [branch] H ~~~ 元のブランチの履歴を残しておくには新しいブランチを作っておく ### [example] 2つ前のコミットメッセージを書き換える ~~~ git rebase -i HEAD~2 変更したい行のpickをeditに変更 git commit --amend -m "message" git rebase --continue git push -f origin master # 上書きに注意 ~~~ ## cherry-pick ~~~ git cherry-pick {commit} ~~~ ## stash ~~~ git stash git stash pop git stash clear git stash list ~~~ ## worktree ~~~ git worktree add {worktree} {branch} git worktree prune # worktree を削除後実行 git worktree list # list worktrees ~~~ ## submodule ### add submodule ~~~ git submodule add URL [dir] ~~~ ### update submodule ~~~ git submodule init git submodule update ~~~ or ~~~ git submodule update --init --recursive ~~~ ### remove submodule ~~~ git submodule deinit [-f] {module} git rm {module} ~~~ ### [example] change submodule URL ~~~ git submodule deinit third_party/{module} git rm third_party/{module} git commit -m "remove submodule {module}" cd third_party git submodule add --force URL [dir] git commit -m "add submodule {module}" cd .. ~~~ ## diff ~~~ git diff [--binary] from_commit [to_commit=HEAD] > diff_file git diff [--cached] [] ~~~ ## apply ~~~ git apply [--check] diff_file ~~~ or ~~~ patch -p1 < diff_file ~~~ ## Links - [Git](https://git-scm.com/) - [GitHub](https://github.com/) - [GitLab](https://about.gitlab.com/) - [サルでもわかるGit入門](https://www.backlog.jp/git-guide/) - [TortoiseGit](https://tortoisegit.org/) - [git manual](https://git.github.io/htmldocs/git.html) - [GitHub GIT CHEAT SHEET](files/github-git-cheat-sheet.pdf) git-svn ======================================================= git | git-svn | svn ----------------------------|------------------------------|----------------------------------- git clone URL | git svn clone URL | svn checkout URL git fetch | git svn fetch | git pull | git svn rebase | svn update git commit | | git push | git svn dcommit | svn commit git add FILE/DIR | | svn add FILE/DIR git rm FILE | | svn rm FILE/DIR git rm -r DIR | | git mv SRC DST | | svn mv SRC DST git checkout -- FILE | | svn revert FILE git checkout BRANCH | | svn switch URL git branch BRANCH | git svn branch BRANCH | svn copy URL URL git tag TAG | git svn tag TAG | svn copy URL URL git merge BRANCH | | svn merge -r REV1:REV2 URL git remote -v | git svn info | git status | | svn status git log | | svn log git diff | | svn diff ## git-svn ~~~ git svn clone file:///x/path/myproj #
